# 墨迹 (Moji) 用户手册

## 1. 产品概述

### 1.1 墨迹是什么

墨迹是一款面向大学生和教师的 **Git 版本管理写作工具**。它将 Git 这一专业开发者使用的版本控制系统，包装成一个简单直观的图形化写作环境。用户通过"章节文件夹"管理文档内容，每次修改自动被 Git 追踪记录，随时可以回溯到任意历史版本，也可以通过分支实现师生之间的反馈协作。

**核心场景**：
- 学生在 main 分支上撰写毕业论文、课程报告
- 教师创建 feedback 分支对文章进行批注修改
- 学生切换分支查看教师反馈，确认后合并

**一句话概括**：像写代码一样写文章，每一次修改都有据可查。

### 1.2 Git 简介

#### Git 的诞生

Git 诞生于 2005 年，由 **Linus Torvalds**（Linux 操作系统之父）创造。在此之前，Linux 内核社区一直使用一个叫做 BitKeeper 的商业版本控制工具。当 BitKeeper 公司收回免费使用权后，Linus 决定自己动手写一个分布式版本控制系统 —— 仅仅用了 **10 天**，Git 的第一个可用版本就诞生了。

Linus 给 Git 设定了几个核心目标：
- **速度快**：能在几秒内处理 Linux 内核那海量的代码提交
- **分布式**：每个人电脑上都有完整的仓库副本，不依赖中央服务器
- **数据完整性**：每次提交都有唯一的 SHA-1 哈希指纹，篡改会被立刻发现
- **支持非线性开发**：分支的创建和合并极其轻量

#### 为什么 Git 适合写作

Git 最初为代码而生，但它的核心能力与写作高度契合：

| Git 的代码能力 | 对应写作场景 |
|:---|:---|
| 追踪每次代码修改 | 记录每次文档改动 |
| `git log` 查看提交历史 | 回顾写作历程 |
| `git diff` 对比版本差异 | 查看文章具体改了什么 |
| `git checkout` 恢复旧版本 | 回退到三天前的内容 |
| 分支 (branch) | 教师独立批注，不影响学生原文 |
| 合并 (merge) | 将教师反馈融入正文 |

市面上大多数写作工具（Word、LaTeX、Markdown 编辑器）只保存"当前最新版本"。一旦你删掉一段文字然后保存，它就永远消失了 —— 除非你手动备份了无数个 `论文_v1.docx`、`论文_v2.docx`、`论文_最终版_v3.docx` 这样的文件。

Git 则自动帮你记录每一次提交的完整快照。你可以随时穿越回到任何一个版本，查看当时写了什么、谁改了什么、什么时候改的。

#### Git 的几个核心概念

**仓库 (Repository)**：一个被 Git 管理的目录。里面有一个隐藏的 `.git` 文件夹，存储了所有的版本历史。在墨迹中，你打开的"工作区"就是一个 Git 仓库。

**提交 (Commit)**：一次"保存快照"的操作。每次 commit 都会生成一个唯一的哈希值（如 `a58d14e`），用来标识这个版本。每次 commit 必须附带一段 message，说明这次改了什么。在墨迹中，commit message 会自动生成，你也可以自定义。

**分支 (Branch)**：一条独立的开发线。默认分支叫 `main`（主干）。你可以在任意时刻从 main 分出一条新分支，在上面的修改完全不影响 main。这在教学场景中特别有用：
- 学生在 `main` 分支写原文
- 教师从 main 分出 `feedback-张三` 分支，在上面批改
- 学生切回 main 查看原文，切到 feedback 分支查看批注
- 确认无误后，将 feedback 分支合并回 main

**合并 (Merge)**：将一个分支的修改并入另一个分支。比如教师改完后，学生把 `feedback-张三` 合并到 `main`，教师的所有批改就融入正文了。

**回退 (Rollback / Checkout)**：将某个文件或文件夹恢复到历史版本。写论文时常有这样的场景：改了半天发现三天前的版本更好。有了 Git，你只需要找到那次 commit，一键回退。

#### 墨迹如何封装 Git

你不需要学 Git 命令行。墨迹将所有 Git 操作都做成了可视化的按钮和面板：

| 你做的事 | 墨迹执行的 Git 命令（你看不到） |
|:---|:---|
| 点击"提交变更" | `git add -A` + `git commit -m "..."` |
| 点击"回退章节到此版本" | `git checkout <hash> -- <chapter>` |
| 点击"新建分支" | `git branch <name>` + `git switch <name>` |
| 切换分支下拉框 | `git switch <name>` |
| 点击"合并到当前" | `git merge <name>` |
| 点击"推送到远程仓库" | `git push -u origin main` |

### 1.3 墨迹与其它写作工具的对比

| 特性 | 墨迹 | Word | LaTeX | 石墨/腾讯文档 |
|:---|:---|:---|:---|:---|
| 版本历史 | 完整 Git 历史，可回退 | 需手动另存为 | 需手动备份 | 有限的历史记录 |
| 分支协作 | 原生支持，教师独立批注 | 仅"修订"模式 | 手动合并 | 仅评论/建议 |
| 离线使用 | 完全离线，本地仓库 | 支持 | 支持 | 需联网 |
| 远程同步 | 推送到 GitHub 等 | OneDrive | 手动 | 自动云端 |
| 文件格式 | 支持任意格式 | .docx | .tex | 在线文档 |
| 学习门槛 | 低（图形化封装） | 低 | 高 | 低 |
