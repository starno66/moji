# 分支管理与教师修改功能 — 设计规格

## 目标

面向大学生和教师，利用 git 分支实现教师独立修改学生文章的能力。教师在专属分支上修改提交，不影响学生原文 (main)；修改完成后可选择合并，学生可切换分支对比查看。

## 改动文件

- `gitmanager.h/.cpp` — 新增分支操作方法
- `mainwindow.h/.cpp` — 新增分支栏 UI 和交互
- `mainwindow.ui` — 新增分支栏控件

## 分支栏 UI

右侧 Commit 面板顶部，Commit 历史标题下方新增一行分支栏：

```
┌──────────────────────────────────────────┐
│ 分支: main  [新建分支] [切换分支] [合并]  │
│                                          │
│ ── Commit 历史 ──                        │
│ ...                                      │
```

分支名为下拉框，列出所有本地分支，直接选择即可切换。

## GitManager 新增方法

| 方法 | 说明 |
|------|------|
| `listBranches()` | `git branch` | 列出所有本地分支 |
| `createBranch(name)` | `git branch <name>` | 创建新分支（不切换） |
| `switchBranch(name)` | `git switch <name>` | 切换到指定分支 |
| `mergeBranch(name)` | `git merge <name>` | 将指定分支合并到当前分支 |

## 交互流程

### 教师流程
1. 打开工作区 → 确认在 main 分支
2. 点击"新建分支" → 输入 `feedback-张三`
3. 自动切换到新分支
4. 用 Word 修改学生文件 → 墨迹检测变更 → 提交 commit（附修改说明）
5. 重复修改提交
6. 可选：切回 main → 点击"合并" → 选择 `feedback-张三` → 确认合并

### 学生流程
1. 打开工作区 → 在 main 分支看原文
2. 切换分支下拉框 → 选 `feedback-张三` → 查看教师修改
3. commit 历史显示所有教师修改记录
